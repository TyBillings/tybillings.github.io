Basic Usage¶

Installation¶

Prerequisites:

  • python=^3.12

  • poetry=^1.8

  • cargo=^1.82.0

To install the free-range-zoo library:

# Clone the repository
git clone git@github.com:oasys-mas/free-range-zoo.git
cd free-range-zoo

# Install dependency packages
poetry install [[--with models]] # NOTE: `--with model` option is intended only for internal OASYS-MAS use.

# Verify that CUDA drivers are present and working
# NOTE: Only necessary if you are expecting to run on CUDA / GPU
python -c "import torch; print(torch.cuda.is_available())"

At the moment only Python 3.12 is directly supported, however it is likely that free-range-zoo is also compatible with 3.11 and 3.13.

Initializing Environments¶

Our environments differ slightly from pettingzoo in two aspects. Our environments operate in vectorized batches. The intention of this is to take advantage of batched computation provided by SIMD architectures, and allow for multithreaded operations to make environment transitions faster. Additionally, our environments use standard Configuration objects to organize environment parameters.

Importing Environments¶

The method for importing environments for utilization is identical to pettingzoo.

# free-range-zoo
from free_range_zoo.envs import wildfire_v0

# pettingzoo
from pettingzoo.butterfly import pistonball_v6

Configuring Environments¶

In order to make environments more flexible and parameterize transitions, free-range-zoo utilizes Configuration classes to group parameters for the environment. Each environment’s respective Configuration objects and structure are documented in their Specification page, and implementations are included in envs/<environment_name>/env/structures/configuration.py.

For simplicity, generators for configurations have been included with the competition materials.

Then we import the default configuration for the environment. defaultConfiguration is an instance of the Configuration class whose attributes are groups of parameters for the environment.

In rideshare, GridConfiguration, PassengerConfiguration, AgentConfiguration, and RewardConfiguration are these seperate attributes. Each of these are a dataclass which contain parameters. To see the details of these configurations go to the environment page in this doc.

from free_range_zoo.envs.rideshare import defaultConfiguration

Then we initialize the environment. Here we show the parallel_api but we also support the aec_api from Pettingzoo/Gymnasium.

import torch
from free_range_zoo.envs import rideshare_v0
from free_range_zoo.envs.rideshare import defaultConfiguration

DEVICE= torch.device('cuda:0') #to use the first GPU on this machine.

configuration = defaultConfiguration.to(DEVICE) #move our config to the device

env = rideshare_v0.parallel_env(
    max_steps = 30, #parameter across all environments for truncations
    parallel_envs=5, #how many independent environments are we running?
    device= DEVICE,
    configuration=configuration
)

Interacting With Environments¶

Environments can be interacted with using a similar interface to Pettingzoo. Here we show a simple way of generating actions if your policy supports batches (and you want to learn in batches), how you can learn multiple policies from independent environments (helpful when using differing seeds as random restarts), and gymnasium style action space sampling.

import torch
from free_range_zoo.envs import rideshare_v0
from free_range_zoo.envs.rideshare import defaultConfiguration

DEVICE= torch.device('cuda:0') #to use the first GPU on this machine.

configuration = defaultConfiguration.to(DEVICE) #move our config to the device

env = rideshare_v0.parallel_env(
    max_steps = 30, #parameter across all environments for truncations
    parallel_envs=5, #how many independent environments are we running?
    device= DEVICE,
    configuration=configuration
)

#seed can be a list of length 5 here or a single value
observations, info = env.reset(seed=42)

while not torch.all(env.finished):
    
    #--------- Actions --------

    #for some policy dictionary {'agent': policy([obs])-> [actions]}
    agent_actions = {agent: 
        policies[agent].act(observations[agent])
    for agent in env.agents}

    #or you can have independent policies learning in parallel by doing {'agent': [policy_0, policy_n]}
    agent_actions = {agent:
        torch.stack(
            [
                policies[agent][batch_index].act(obs)
            for batch_index, obs in enumerate(observations[agent])]
    for agent in env.agents}

    #or sampling
    agent_actions = {agent:
        torch.stack([torch.tensor(space.sample(), device=DEVICE) for space in env.action_space(agent)])
    for agent in env.agents}

    #-----------------------------

    observations, reward, terminations, truncations, info = env.step(agent_actions)
